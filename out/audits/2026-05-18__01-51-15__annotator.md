# Task Audit: Metaspace Exhaustion Hunt in a Hot-Reloading Java Plugin Daemon

**Verdict: Ready**
The task now cleanly surfaces its constraints, the oracle passes on the current checksum, and I did not find a broken verifier, hidden hint, or realistic cheap bypass that would make it unfit to ship.

---

## What the Task Does
This task ships a Java daemon, `textd`, that hot-swaps plugin revisions while serving work on a fixed four-thread pool. The core bug is still a real multi-step lifetime leak: stale per-thread worker bindings pin old generations across reloads, and the last thread-local binding survives session close unless shutdown clears worker-local state. The task now also makes the runtime-layer condition explicit: the service uses a project-local JVM profile in `config/jvm.cfg`, and the shipped CI profile disables explicit GC, so the full fix requires both daemon-side cleanup and correcting the runtime profile without mutating the validation harness. The expected solution is therefore broader than a one-line code fix, but still one coherent debugging task around reclaiming retired generations under repeated reloads.
