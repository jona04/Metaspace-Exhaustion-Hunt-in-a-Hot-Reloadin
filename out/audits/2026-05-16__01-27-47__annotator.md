# Task Audit: Metaspace Exhaustion Hunt in a Hot-Reloading Java Plugin Daemon

**Verdict: Ready**
The prompt and agent-visible replay surface now expose the real invariant, and the remaining issues are polish-level validation-flow concerns rather than must-fix fairness problems.

---

## What the Task Does
This task ships a Java daemon, `textd`, that hot-swaps plugin revisions while serving work on a fixed four-thread pool. The real bug is still a multi-part lifetime leak in the daemon state flow: stale worker bindings pin old plugin generations across reloads, and the last thread-local binding survives session close unless shutdown clears worker-local state. On top of that, the CI JVM is launched with explicit GC disabled, so observable class-unload progress only shows up if the agent notices the runtime layer as well as the daemon leak. The expected solve path is therefore broader than a one-line code fix, but it is still one coherent debugging task around reclaiming retired generations after repeated reloads.
